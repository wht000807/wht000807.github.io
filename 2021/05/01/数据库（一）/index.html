数据库学习笔记（一）

第一章：在单一表格中检索数据

  • USE：表示选择一个默认的数据库。
  • SELECT : 选择指定表格中的列。可以包含运算符也可以用AS创建新的列，如果列表中要包含空格则用“ ”括起来。用”,”分隔。SELECT*代表选择当前表格所有列
  • WHERE : 表述筛选条件。可以包含运算符“AND、OR、NOT”
  • IN：表示多个条件 IN(‘ ’，‘ ’)。可以与NOT连用
  • BETWEEN : 表示>=a AND <=b。 等价于 BETWEEN a AND b 。
  • **LIKE : **筛选字符串。 LIKE‘%a’表示代替n位 LIKE’_b’表示代替一位
  • REGEXP : 正则表达式。 REGEX ‘ abc’ 表示字符串中包含abc的所有结果 “^abc”表示限制abc开头，”abc$”表示限制abc结尾 ，‘a|b|c’表示包含a或b或c，‘[abc]d’表示ad或bd或cd，‘[a-h]e’表示(a-h)+e
  • NULL : WHERE IS NULL 表示条件为空的结果
  • ORDER BY : 表示按条件排序,可以包含多个列。 ORDER BY list1,list2 DESC表示降序
  • LIMIT：LIMIT a 表示限制数量，LIMIT a,b 表示从a开始获取b个数据。LIMIT子句要放在最后

第二章：在多张表格中检索数据

内连接 INNER JOIN

FROM table1 t1
JOIN table2 t2
	ON t1.id = t2.id

跨数据库连接

只需要在JOIN连接的时候加上位置的前缀

FROM table1 t1
JOIN sql2.table2 t2
	ON t1.id = t2.id

自连接

我们可以将数据库自己的列相连接来显示关系结构图

FROM table1 t1
JOIN table1 t2
	ON t1.id1 = t2.id2

多表连接

我们可以使用多个JOIN-ON来实现多表连接

FROM table1 t1
JOIN table2 t2
	ON t1.id = t2.id
JOIN table3 te
	ON t1.id = t3.id

复合连接条件

有的时候单列无法作为唯一标识，因此我们用AND标识符在ON后面来实现多个列的连接

FROM table1 t1
JOIN table2 t2
	ON t1.id1 = t2.id1
	AND t1.id2 = t2.id2

隐式连接(不建议使用)

1中的式子可以替换为：

FROM table1 t1,table2 t2

WHERE t1.id = t2.id

注意：如果忘记打WHERE 数据会交叉

外连接 OUTER JOIN

当不仅仅想要对应的数据时我们会使用外连接来列出全部数据

SELECT*
FROM table1 t1
LEFT/RIGHT JOIN table2 t2
	ON t1.id = t2.id

注意：LEFT JOIN返回t1中的所有数据 ,不管t2中有没有相匹配的数据。

​ RIGHT JOIN 返回t2中的所有数据,不管t1中有没有相匹配的数据。

多表外连接

如果想展现多表的全部，我们要使用多重LEFT/RIGHT JOIN 。

SELECT*
FROM table1 t1
LEFT/RIGHT JOIN table2 t2
	ON t1.id = t2.id
LEFT/RIGHT JOIN table3 t3
	ON t1.id = t3.id

注意：为了代码的可读性，在多表外连接时我们尽量只使用LEFT JOIN！

自外连接

在自连接的前面加个LEFT即可返回所有的数据。

USING子句

如果两个表中有列的名称是完全一样的

SELECT *
FROM table1 t1
JOIN table2 t2
	--ON t1.id = t2.id
	USING(id)
JOIN table t3
	USING(name)
SELECT *
FROM table1 t1
JOIN table2 t2
	--ON t1.id = t2.id AND t1.name = t2.name
	USING(id,name)

注意：USING关键字只能在不同表中列名完全一样的情况下使用

自然连接

自动连接两个数据库相匹配的列

SELECT *
FROM table1 t1
NATURAL JOIN table2 t2

交叉连接

在一个型号表中，如果我们需要将所有的型号和所有的颜色组合，我们可以使用交叉连接

SELECT *
FROM table1 t1
CROSS JOIN table2 t2
SELECT *
FROM table1 t1,table2 t2

联合

UNION关键字可以合并多段查询的记录

USE sql_store;
SELECT 
	customer_id,
	first_name,
	points,
    'Bronze' AS type
FROM customers
WHERE points<2000
UNION
SELECT 
	customer_id,
	first_name,
	points,
    'Siliver' AS type
FROM customers
WHERE points BETWEEN 2000 AND 3000
UNION
SELECT 
	customer_id,
	first_name,
	points,
    'GOLD' AS type
FROM customers
WHERE points>3000

第四章：汇总数据

聚合函数

--MAX()
--MIN()
--AVG()
--SUM()
--COUNT()

SELECT 
	MAX(invoice_total)
	MIN(invoice_total)
	AVG(invoice_total)
	SUM(invoice_total)
	COUNT(invoice_total)
FROM invoices
WHERE...
--COUNT(invoice_total)计算的是这个列里面非空的元素，
--COUNT(*)计算的列的总数
--DISTINCT关键字排除了列中重复的元素

GROUP BY 子句

--单列分组
SELECT
	client_id,
	SUM(invoice_total) AS total_sales
FROM invoices
GROUP BY client_id

--计算不同client_id总的销售额
--GROUP BY 放在ORDER BY 前面
--多列分组
SELECT
	state,
	city,
	SUM(invoice_total) AS total_sales
FROM invoices i
JOIN clients USING (client_id)
GROUP BY state,city

HAVING子句

用于分组后筛选数据，与WHERE的用法基本一致。

SELECT
	client_id,
	SUM(invoice_total) AS total_sales
FROM invoices
GROUP BY client_id
HAVING total_sales >500

WHERE 与 HAVING 的区别：前者用于分组行前筛选，后者用于分组后

ROLLUP运算符（只适用MySql）

我们再GROUP BY 后面加入WITH ROLLUP 来得到汇总的结果集

SELECT
	client_id,
	SUM(invoice_total) AS total_sales
FROM invoices
GROUP BY client_id WITH ROLLUP
SELECT
	state,
	city,
	SUM(invoice_total) AS total_sales
FROM invoices
JOIN clients USING(client_id)
GROUP BY state,city WITH ROLLUP
--多列分组运用rollup运算符时会得到每个组及整个结果集的汇总值

第六章：编写复杂查询

子查询

--例1：
SELECT *
FROM products
WHERE unit_price>(
	SELECT unit_price
    FROM products
    WHERE product_id = 3
)

--例2：
SELECT *
FROM employees
WHERE salary>(
	SELECT AVG(salary)
    FROM employees
)

IN 运算符

SELECT *
FROM products
WHERE product_id NOT IN(
		SELECT DISTINCT product_id
		FROM order_items 
        )

子查询VS连接

--子查询获得购买产品id为3的顾客信息
SELECT DISTINCT c.customer_id,c.first_name,c.last_name
FROM orders
JOIN customers c USING(customer_id)
WHERE order_id IN(
SELECT order_id
FROM order_items
WHERE product_id = 3
)
--连接查询获得购买产品id为3的顾客信息
SELECT DISTINCT customer_id,first_name,last_name
FROM customers c
JOIN orders o USING(customer_id)
JOIN order_items oi USING (order_id)
WHERE oi.product_id = 3

ALL关键字

ALL运算符是一个逻辑运算符，它将单个值与子查询返回的单列值集进行比较。

--获得大于顾客三发票的所有发票
SELECT *
FROM invoices
WHERE invoice_total>(
SELECT MAX(invoice_total)
FROM invoices
WHERE client_id = 3
)

--等价于
SELECT *
FROM invoices
WHERE invoice_total> ALL(
SELECT invoice_total
FROM invoices
WHERE client_id = 3
)
条件 描述
c > ALL(…) c列中的值必须大于要评估为true的集合中的最大值。
c >= ALL(…) c列中的值必须大于或等于要评估为true的集合中的最大值。
c < ALL(…) c列中的值必须小于要评估为true的集合中的最小值。
c <= ALL(…) c列中的值必须小于或等于要评估为true的集合中的最小值。
c <> ALL(…) c列中的值不得等于要评估为true的集合中的任何值。
c = ALL(…) c列中的值必须等于要评估为true的集合中的任何值。

ANY关键字

--选择至少有两张发票的客户
SELECT *
FROM clients
WHERE client_id IN (
    SELECT client_id,COUNT(*) AS number
	FROM invoices
	GROUP BY client_id
	HAVING number >=2
)

SELECT *
FROM clients
WHERE client_id = ANY (
    SELECT client_id,COUNT(*) AS number
	FROM invoices
	GROUP BY client_id
	HAVING number >=2
)

注意：=ANY 等价于IN。

相关子查询

--在所有员工中，计算他们办公室的平均工资并返回工资大于平均工资的员工
SELECT *
FROM employees e
WHERE salary >(
	SELECT AVG(salary)
    FROM employees
    WHERE office_id = e.office_id
)

相关子查询与非相关子查询的区别：

非相关子查询的执行不依赖与外部的查询。
执行过程：
（1）执行子查询，其结果不被显示，而是传递给外部查询，作为外部查询的条件使用。
（2）执行外部查询，并显示整个结果。　　
非相关子查询一般可以分为：返回单值的子查询和返回一个列表的子查询。

相关子查询的执行依赖于外部查询。多数情况下是子查询的WHERE子句中引用了外部查询的表。
执行过程：
（1）从外层查询中取出一个元组，将元组相关列的值传给内层查询。
（2）执行内层查询，得到子查询操作的值。
（3）外查询根据子查询返回的结果或结果集得到满足条件的行。
（4）然后外层查询取出下一个元组重复做步骤1-3，直到外层的元组全部处理完毕。 　　

总结：

非相关子查询是独立于外部查询的子查询，子查询总共执行一次，执行完毕后将值传递给外部查询。

相关子查询的执行依赖于外部查询的数据，外部查询执行一行，子查询就执行一次。
故非相关子查询比相关子查询效率高。

EXISTS运算符

--选择有发票的客户
--方法1：
SELECT *
FROM clients C
WHERE client_id IN(
    SELECT DISTINCT client_id
    FROM invoices
)
--方法2：
SELECT *
FROM clients C
WHERE EXISTS(
    SELECT  client_id
    FROM invoices
    WHERE client_id = c.client_id
)

EXISTS 没有返回的结果只返回TRUE/FALSE，而IN有返回的列表，故当返回的数据较大时，IN的效率没有EXISTS高。

SELECT子句中的子查询

SELECT 
	invoice_id,
    invoice_total,
    (SELECT AVG(invoice_total)
		FROM invoices) AS invoice_average,
	invoice_total-invoice_average
FROM invoices

FROM子句中的子查询

SELECT *
FROM(
    SELECT
		client_id,
		name,
		(SELECT SUM(invoice_total)
    		FROM invoices
    		WHERE client_id = c.client_id) AS total_sale,
    	(SELECT AVG(invoice_total)FROM invoices) AS average,
    	(SELECT total_sale -average) AS differences
	FROM clients c    
)AS sales_summary
WHERE total_sales IS NOT NULL

第七章：MySql基本函数

数值函数

ROUND(a,b)  --四舍五入小数a，保留b位
CEILING(c)  --返回大于或等于c的最小整数
FLOOR(d)    --返回小于或等于d的最大整数
ABS(e)      --计算e的绝对值
RAND()      --返回0-1直接的随机浮点数

字符串函数

LENGTH（''）    --返回字符串的长度
UPPER('')      --返回字符串的大写形式
LOWER('')      --返回字符串的小写形式
LTRIM('')      --移除字符串左侧的空白字符或其他预定义字符
RTRIM('')      --移除字符串右侧的空白字符或其他预定义字符
TRIM('')       --删除所有前导或者尾随空格
LEFT('',a)     --返回字符串从左数a位
LEFT('',a，b)  --返回字符串从第a位开始数b位
RIGHT('',a)    --返回字符串从右数a位
LOCATE('a','b')--从1开始，返回字符b中第一个出现a的位置，没有则返回0
REPLACE('a','b','c')--将字符串a中的b元素换成c元素
CONCAT('a','b') --连接a,b两个字符串

日期函数

NOW()/CURDATE()/CURTIME()           --返回当前日期+时间/当前日期/当前时间
YEAR()/MONTH()/DAY()/MINUTE()       --返回时间的年份/月份...
YEARNAME()/MONTHNAME()...           --返回年/月...的字符串形式
EXTRACT(timea FROM timeb)           --返回从时间b中提取时间a

格式化日期和时间

DATE_FORMAT(time,'%a%b')   --格式化时间%a	缩写星期名
TIME_FORMAT(time,'%a%b')

%c	月，数值
%D	带有英文前缀的月中的天
%d	月的天，数值(00-31)
%e	月的天，数值(0-31)
%f	微秒
%H	小时 (00-23)
%h	小时 (01-12)
%I	小时 (01-12)
%i	分钟，数值(00-59)
%j	年的天 (001-366)
%k	小时 (0-23)
%l	小时 (1-12)
%M	月名
%m	月，数值(00-12)
%p	AM 或 PM
%r	时间，12-小时（hh:mm:ss AM 或 PM）
%S	秒(00-59)

%T	时间, 24-小时 (hh:mm:ss)

计算时间和日期

DATE_ADD(time,INTERVAL 1 DAY)   --增加日期
DATE_SUB(time,INTERVAL 1 DAY)   --减去时间
DATEDIFF(timea,timeb)           --返回时间a-时间b的天数（存在负值）
TIME_TO_SEC(time)               --返回从0点开始计时的秒数

IFNULL函数和COALESCE函数

--IFNULL函数：当括号中第一个元素为空则替换成第二个元素
SELECT 
	order_id,
	IFNULL(shipper_id,'asd')
FROM orders

--COALESC函数：当括号第一个元素为空则替换成第二个，如果第二个为空则替换成第三个
SELECT 
	order_id,
	COALESC(shipper_id,comment,'...')
FROM orders

IF函数

--IF(expression,first,second)   IF表达式为真返回第一个，为假返回第二个

SELECT
	order_id,
	order_date,
	IF(YEAR(order_date)=YEAR(NOW()),'Active','Archived')
FROM orders

CASE运算符

--在有多个测试表达式且想要针对每个表达式返回不同值的时候可以使用CASE运算符

SELECT
	order_id,
	CASE	
		WHEN YEAR(order_date) = YEAR(NOW()) THEN 'Active'
		WHEN YEAR(order_date) = YEAR(NOW())-1 THEN 'Last Year'
		WHEN YEAR(order_date) < YEAR(NOW())-1 THEN 'Archived'
		ELSE 'Future'
	END AS category
FROM orders
